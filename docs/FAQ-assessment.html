How do I add a caption to a summary output table?

pander() can be used to output a summary table with a caption. Use the caption=“XXX” argument. For example:

library(tidyverse)
library(pander)

mtcars %>% 
  group_by(cyl) %>% 
  summarise(mean_mpg = mean(mpg)) %>% 
  pander(caption = "Mean miles per gallon by cyl")
Mean miles per gallon by cyl
cyl mean_mpg
4 26.66
6 19.74
8 15.1



What does it mean if RStudio prints out “e-” next to a number?

If R prints “1.4e-4”, in the output this actually means “\(1.4 \times 10^{-4}\)”, or “0.00014”. It is a way of printing out very small (or very large) numbers.

“e-4” means “move the decimal point 4 places to the left”. See this by executing the command below – it should return “0.00014”.

1.4e-4
> [1] 0.00014

So:

  • “e-5” means move the decimal point 5 places to the left (e.g., 2.5e-5 is “\(2.5 \times 10^{-5}\)”, or 0.000025)
  • “e-10” means move the decimal point 10 places to the left (e.g., 2.5e-10 is “\(2.5 \times 10^{-10}\)”, or 0.00000000025).
  • “e5” means move the decimal point 5 places to the right (e.g., 2.5e5 is “\(2.5 \times 10^{5}\)”, 250000).

Do all the elements of my figure and and table need to conform 100% to APA style (e.g., italicisations of text in the figure title etc.)?

This would be difficult to achieve given what has been covered so far on the course. For example, you have not been shown how to italicize text embedded in figures, and so would not be expected to do this. To clarify, the requirement in the assignment brief to make your figure and table “of the standard that you would expect to see in an academic journal” does not mean that they have to conform exactly to the formatting guidelines stated in the APA publication manual. It is more important to consider what data you will show in your table and figure, to design code that will do this effectively, and for the appearance of the table and figure to be clear in your report.


If you still would like to know how to italicize titles in your figure, then read on.


As covered in week 17, the symbols * and _ can be placed around words in the text of an Rmd file to italicize them or make them formatted in bold. These symbols, however, don’t work the same way in code chunks as they do in the text of an Rmd file.


If you want to italicize a figure title, use expression() and italic() when specifying the title in your code chunk. For example, you can add this code to your ggplot:

+ labs(title = expression( italic("your title here") ))


Here’s an example:

mtcars %>% 
  ggplot(aes(x = hp, y = mpg)) +
  geom_point() +
  labs(title = expression( italic("Miles Per Gallon vs. Horse Power")) )


How can I find out more about a particular function, so I can play around with its settings etc.?

For any function in R, if you execute ?function_name in your code or at the console, then R will handily load a help file with details of the function and all its options. Try, for example, running ?geom_point



Multicollinearity: If two predictor variables have a correlation greater than r = 0.9, then multicollinearity is a problem. Is the same also true if the correlation exceeds r = -0.9?

Yes, multicollinearity is a problem when there is a very strong relationship between predictor variables. Correlations that exceed 0.9 or -0.9 are considered problematic.



Should R code be visible in my html report?

No code needs to be included in the html report, and can be prevented from appearing in your report by using the commands covered in the “Using Rmd files” Powerpoint (in Building Models 2).


Covered there were two helpful relevant commands that could be used at the top of a code chunk: 1) include = FALSE if you want to prevent code and associated output from appearing in the html, and 2) echo = FALSE to show the output (e.g., a figure or a table), but not the code that generated it. You marker will go through the Rmd file you submit to see the code you used for your analysis, table and figure.



How might the html report look?

A typical html report will consist of a title, your SRN, a figure, a table, with text interspersed explaining the multiple regression analysis you have performed and with statistics presented in APA style. You may lay out the content of the report as you see fit.



Do I need an introduction section?

As stated at the outset of the brief, “you are required…to submit a short report of your findings, in the style similar to that which you might expect to see in the results section of an academic journal”. In my sessions I mentioned that it will be a good idea to devote a sentence or two at the outset to provide context (e.g., mentioning that the dataset concerns sport and cheating) and describing what you will look at in the report (e.g., you are interested in the relationship between x, y and z etc.), so that what follows makes sense to a reader.
